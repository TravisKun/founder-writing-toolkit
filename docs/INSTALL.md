# FWT Installation Guide (Windows)

## Prerequisites

- Windows 10/11
- [Python 3.10+](https://python.org/downloads/) — check "Add to PATH" during install
- [Git](https://git-scm.com/)
- Chrome or Edge (any recent version)
- An [Anthropic API key](https://console.anthropic.com/settings/api-keys)

---

## Option A — One-liner install (PowerShell)

Open **PowerShell** (not cmd) and run:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass; & "$env:USERPROFILE\fwt\scripts\install.ps1"
```

If you have not cloned the repo yet, the script will ask for the GitHub URL and clone it first.

If the repo is already on your machine, `cd` to it and run:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass; .\scripts\install.ps1
```

The script will:
1. Check that `git` and `python` are installed
2. Create `fwt-backend/.venv` if missing
3. Install Python dependencies from `requirements.txt`
4. Prompt for your `ANTHROPIC_API_KEY` (hidden input) and create `fwt-backend/.env`
5. Start the backend on `http://localhost:8000`
6. Print the 3 steps to load the extension

---

## Option B — Manual setup

```powershell
# 1. Create venv
cd fwt-backend
python -m venv .venv
.\.venv\Scripts\activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Create .env
#    Open fwt-backend/.env in any text editor and add:
#      ANTHROPIC_API_KEY=sk-ant-...
#      ANTHROPIC_MODEL=claude-sonnet-4-6

# 4. Start backend
uvicorn main:app --reload --port 8000
```

## Where to paste your API key

Edit `fwt-backend/.env`:

```
ANTHROPIC_API_KEY=sk-ant-YOUR_KEY_HERE
ANTHROPIC_MODEL=claude-sonnet-4-6
```

- Get a key at: https://console.anthropic.com/settings/api-keys
- The backend exits immediately on startup if the key is missing.
- Default model: `claude-sonnet-4-6` (override by changing `ANTHROPIC_MODEL`).

## Load the extension in Chrome / Edge

1. Open `chrome://extensions` (Chrome) or `edge://extensions` (Edge)
2. Enable **Developer mode** (toggle in the top-right corner)
3. Click **Load unpacked** and select the `extension/` folder inside the repo
4. The FWT icon appears in the toolbar

### First-time panel open

Without a service worker, the very first click on the toolbar icon may not open the panel.

**Fix:** Right-click the FWT icon → **"Open side panel"** (or find it in the extensions menu).

After that, icon-click opens the panel automatically.

## Default URLs

| Service | URL |
|---------|-----|
| Backend health | `http://localhost:8000/health` |
| Interactive API docs | `http://localhost:8000/docs` |
| Backend port | `8000` (hardcoded in `manifest.json` host_permissions) |

---

## Start backend in future sessions

Run `scripts\start_backend.ps1` from the repo root — no need to re-run the full install:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass; .\scripts\start_backend.ps1
```

---

## Offline / mock mode

To use the extension without a running backend (no API key needed):

1. Open `extension/shared/provider.js`
2. Change `const USE_BACKEND = true;` to `const USE_BACKEND = false;`
3. Reload the extension in `chrome://extensions`

Variants will be generated by a local mock — useful for testing the UI without an API call.
